Unidad 4.2: Modelos de Colaboración

1. Shared Repository (Modelo Compartido)

Todos los desarrolladores tienen permisos de escritura (git push) directos sobre el repositorio oficial. Ideal para Startups y equipos pequeños.

# Flujo tipico de trabajo diario:
> git pull origin main # Bajar cambios de los demas
> git commit -m "feat: nuevo navbar"
> git push origin main # Subir directamente al server oficial

2. Forking Workflow (Zero Trust)

Nadie toca el repositorio oficial (Upstream). Se hace una fotocopia (Fork), se trabaja localmente, y se solicita una revisión formal (Pull Request).

# Flujo de trabajo en modo Forking:
> git pull upstream main # Sincronizar con la empresa
> git switch -c feat/mi-tarea # Crear rama aislada
> git commit -m "feat: tarea completada"
> git push origin feat/mi-tarea # Subir a MI cuenta (Fork)

Analogía Pedagógica

El modelo compartido es como una Pizarra en la oficina: todos dibujan a la vez, lo que puede causar borrones accidentales. El modelo Forking es como una Editorial de Periódico: escribes tu nota en tu PC, y el Jefe de Edición (Reviewer) aprueba si se publica o no en el diario oficial.
